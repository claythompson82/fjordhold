# Fjordhold Roadmap

## v0.1.0 — Hearthfire Prototype

Goal: prove the core game loop.

Features:

- playable Frostpine Fjord map
- player movement and camera
- gathering wood/stone/food/iron
- inventory and hotbar
- modular building placement
- roof fade / interior reveal
- workbench crafting
- basic tools, weapon, shield, armor
- one enemy faction
- one night raid loop
- local save/load
- initial UI polish

Release quality:

- playable from README instructions
- no major boot errors
- basic screenshots/GIF possible
- original project identity visible

## v0.2.0 — Builder’s Hearth

Goal: deepen building satisfaction.

Potential features:

- more building pieces
- rotation polish
- repair hammer
- structure damage states
- improved placement preview
- more furniture/decor
- comfort/hearth bonus prototype
- better roof zone detection

## v0.3.0 — Raid Night

Goal: make defense more exciting.

Potential features:

- better enemy AI
- enemy target priorities
- multiple raid wave tables
- raid horn/banner polish
- enemy drops
- traps
- watchtower prototype
- structure breach feedback

## v0.4.0 — Forge & Frost

Goal: improve crafting/progression.

Potential features:

- forge station
- iron gear tier
- recipe unlocks
- food buffs
- armor stats
- weapon differences
- better item icons

## v0.5.0 — Public Polish Pass

Goal: make the game more shareable.

Potential features:

- sound pass
- particle pass
- title screen
- settings menu
- controls screen
- save slot/reset UI
- screenshots for README
- packaged Windows build experiment

## Backlog Ideas

These are not MVP commitments:

- boss raid captain
- raven guide
- banner/trophy decorations
- skill progression
- minimap
- farming/herbs
- weather hazards
- procedural map variants
- Steam-style desktop packaging

## Hard Scope Boundaries

Do not add before v0.1.0:

- multiplayer
- multiple biomes
- quests
- NPC villages
- online accounts
- monetization
- boats
- large procedural world system

## Project Health Rule

Every milestone should preserve a playable build. Do not allow long-running branches where the game is broken for days.
